Test Collapsed Node Sizing Fix

Issue Description

Problem: When children in a container node are collapsed, the parent containers are not properly resized because collapsed children still contribute their full expanded dimensions to size calculations.

Root Cause

The issue was in the size calculation logic of container nodes:

  • Expanded children: Correctly used their full node.data.width and node.data.height
  • Collapsed children: Incorrectly used their full node.data.width and node.data.height instead of their minimal node.minimumSize.width and node.minimumSize.height

This caused collapsed children to contribute much more space than they should, keeping parent containers oversized.

Fix Applied

Updated the size calculation logic in three container node types:

1. ColumnsNode

// Before (incorrect) const totalChildWidth = visibleChildren.reduce((sum, node) => sum + node.data.width, 0); // After (correct) const totalChildWidth = visibleChildren.reduce((sum, node) => { const nodeWidth = node.collapsed ? node.minimumSize.width : node.data.width; return sum + nodeWidth; }, 0);

2. LaneNode

// Before (incorrect) const totalChildHeight = visibleChildren.reduce((sum, node) => sum + node.data.height, 0); // After (correct) const totalChildHeight = visibleChildren.reduce((sum, node) => { const nodeHeight = node.collapsed ? node.minimumSize.height : node.data.height; return sum + nodeHeight; }, 0);

3. GroupNode

// Before (incorrect) const halfWidth = node.data.width / 2; const halfHeight = node.data.height / 2; // After (correct) const nodeWidth = node.collapsed ? node.minimumSize.width : node.data.width; const nodeHeight = node.collapsed ? node.minimumSize.height : node.data.height; const halfWidth = nodeWidth / 2; const halfHeight = nodeHeight / 2;

Expected Behavior After Fix

✅ Collapsed children contribute minimal size: When a child node is collapsed, it only contributes its minimumSize to parent container calculations

✅ Parent containers resize correctly: Parent containers now properly size themselves based on the actual space needed by their children

✅ No oversized containers: Containers no longer remain oversized when their children are collapsed

✅ Consistent behavior: Both width and height calculations work the same way for collapsed vs expanded children

Technical Details

How Collapsed Nodes Work

When a container node is collapsed:

  1. It stores its expanded size in data.expandedSize
  2. It resizes itself to minimumSize (typically just enough for the header/label)
  3. Its children become invisible but remain in memory
  4. It should contribute only minimumSize to parent container calculations

Why the Fix Works

The fix ensures that:

  • Collapsed children use minimumSize dimensions
  • Expanded children use their full data dimensions
  • Parent containers receive proper size change notifications
  • Size changes cascade up through the container hierarchy
